À propos de cet outil
Gestion de la charge EQOS Energie · Atlantique Etudes

Cet outil permet de gérer la planification et le suivi de la charge des équipes EQOS Energie et Atlantique Etudes. Il centralise les projets, les ressources, les absences et génère les tableaux de reporting RTE.

Tableau de bord
Vue d'ensemble en temps réel

Le tableau de bord est la page d'accueil de l'application. Il affiche des indicateurs clés calculés à partir des données du moment.

Cartes disponibles

CarteContenu

Planning & Gantt
Tâches, jalons, vue temporelle

Le module Planning offre une vue Gantt interactive par projet, permettant de visualiser et gérer les tâches dans le temps.

Tâches

Sélectionner un projet

Choisissez le projet dans la liste déroulante en haut du planning.

Ajouter une tâche

Cliquez sur « + Tâche » — renseignez le nom, la date de début et la date de fin.

Déclarer un jalon

Cochez l'option « Jalon » et entrez la date cible. Les jalons apparaissent en orange dans le Gantt.

✅ Les tâches dont la date de fin est passée sont automatiquement comptabilisées comme « terminées » dans l'avancement de la fiche projet.

Lecture du Gantt

ÉlémentSignification
Barre bleue (navy)Tâche en cours ou à venir
Barre griséeTâche terminée (date fin dépassée)
Losange orange ◆Jalon
Ligne rouge verticaleDate du jour

Saisie des charges
Affecter les heures ressources aux projets

La saisie des charges permet de planifier le temps de travail de chaque ressource sur les projets, semaine par semaine.

Principe de fonctionnement

Projet
→
Ligne de charge
(projet + ressource)
→
Entrée horaire
(date + nb d'heures)

Ajouter une ligne de charge

Sélectionner le projet

Dans la vue charges, choisissez le projet concerné.

Choisir la ressource

Sélectionnez la ressource à partir de la liste des ressources actives.

Saisir les heures

Pour chaque date, entrez le nombre d'heures planifiées. Les valeurs décimales sont acceptées (ex: 3.5h).

⚠️ Les heures passées (antérieures à aujourd'hui) ne sont pas comptabilisées dans les totaux « à venir » de la fiche projet. Seules les heures futures sont affichées dans le récapitulatif.

Tableau RTE
Rapport hebdomadaire de charge — Export Excel

Le tableau RTE est le rapport de charge hebdomadaire transmis à RTE. Il synthétise, pour chaque ressource et chaque type de poste (LA, LS, CONV), les heures planifiées semaine par semaine.

Structure du tableau

LigneContenuEntité
Lignes 1–8En-têtes et capacités (nb de ressources par poste)—
Lignes 9–11Congés/absences LA · LS · CONVEqos Energie
Lignes 12–14Hors-projets LA · LS · CONVEqos Energie
Lignes 15–17Congés/absences LA · LS · CONVAtlantique Etudes
Lignes 18–20Hors-projets LA · LS · CONVAtlantique Etudes
Lignes 21+Heures par projet (une ligne = une ressource × projet)Les deux

Exporter en Excel

Accéder au tableau RTE

Cliquez sur « Tableau RTE » dans la navigation.

Choisir la semaine

Sélectionnez la semaine de début du rapport via le sélecteur de semaine.

Cliquer sur « Export Excel »

Le bouton orange génère le fichier RTE_SEMAINE_XX.xlsx basé sur le template.

Ouvrir dans Excel

Le fichier s'ouvre avec toutes les données pré-remplies. Les formules et graphiques du 2e onglet sont préservés.

✅ L'export utilise le template Template_RTE.xlsx. Les formules et graphiques placés dans ce template sont automatiquement conservés lors de l'export.
ℹ️ La capacité (nombre de ressources par poste) est calculée au moment de l'export, en comptant les ressources présentes dans la base. Si une ressource arrive en cours d'année, ajoutez-la dans le module Ressources avant de lancer l'export.

Questions fréquentes
Aide à la résolution de problèmes
Le tableau de bord affiche un mauvais décompte de projets ▾
Vérifiez que le statut du projet est bien « En cours ». Seuls les projets avec ce statut sont comptabilisés dans le compteur du tableau de bord. Les projets en statut « Offre », « Stand-by » ou « Terminé » ne sont pas inclus.
Le graphique d'heures de la fiche projet est vide ▾
Cela signifie qu'aucune heure n'est planifiée dans les 6 prochains mois pour ce projet. Seules les heures futures (date ≥ aujourd'hui) apparaissent dans le graphique. Vérifiez que les lignes de charge ont bien des dates à venir.
L'export Excel RTE ne contient pas les bonnes semaines ▾
Assurez-vous d'avoir sélectionné la bonne semaine de départ dans le sélecteur avant de cliquer sur « Export Excel ». Le rapport couvre par défaut un nombre de semaines défini dans le template RTE. Si les colonnes paraissent décalées, vérifiez que le template Template_RTE.xlsx commence bien à la semaine 1 de la bonne année.
Les graphiques du 2e onglet Excel sont cassés après export ▾
C'est un comportement connu d'ExcelJS : il peut parfois affecter les objets graphiques intégrés. Si le problème persiste, contactez l'administrateur de l'outil pour utiliser la version Python de l'export (serveur_rte_1.py) qui préserve mieux les graphiques.
Les lignes absences / hors-projets du RTE sont vides ▾
Vérifiez que les absences et hors-projets ont bien été saisis dans les modules correspondants avant de lancer l'export. Ces données doivent être saisies manuellement — elles ne sont pas calculées automatiquement. Les lignes 9–11 (congés Eqos), 12–14 (hors-projets Eqos), 15–17 (congés AE) et 18–20 (hors-projets AE) doivent refléter les saisies correspondantes.
Un jalon n'apparaît pas dans la fiche projet ▾
La fiche projet n'affiche que les jalons à venir (date ≥ aujourd'hui). Si la date du jalon est passée, il n'apparaît plus dans le récapitulatif. Consultez le Gantt pour voir tous les jalons, y compris les passés.
Comment ajouter une nouvelle ressource ? ▾
Accédez au module Ressources depuis la navigation latérale, cliquez sur « + Ressource », renseignez les champs (Nom, Prénom, Entreprise, Poste, Heures hebdo) et enregistrez. La ressource sera immédiatement disponible pour la saisie des charges et sera prise en compte lors du prochain export RTE.
Les données ne sont pas sauvegardées après fermeture du navigateur ▾
L'application utilise un stockage local (LocalStorage). Les données sont conservées dans votre navigateur tant que vous n'effacez pas le cache. Pour une sauvegarde pérenne, utilisez la fonction Export JSON dans les Paramètres, et importez-les en cas de besoin. En mode multi-utilisateurs, assurez-vous que le serveur de données est bien configuré.
